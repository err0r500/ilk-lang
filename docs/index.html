ilk / kli

A two-level data modeling system for domains where data provenance matters.

Traditional schemas validate shape. ilk/kli validates data flow: where does each field come from? Can the response body only contain data from the DB result? Does every event field trace back to the command input?

ilk (.ilk) defines the abstract vocabulary — what concepts exist, their shapes, and flow constraints via @source.
kli (.kli) instantiates that vocabulary — names the concrete entities and maps their field origins.

Neither level holds runtime data. A .kli file is a domain model — a catalog of named entities and their structures. Actual values (UUIDs, timestamps) live in the system that consumes the validated model.

API Endpoint Example

Complete data flow: request → DB method → response, with @source constraints

api-spec.ilk
HttpMethod "GET" | "POST" | "PUT" | "DELETE"

DbMethod {
    name    Concrete<String>
    args    {...}
    returns {...}
}

Endpoint {
    @constraint forall(templateVars(path), v => v in keys(params))
    path    Concrete<String>
    method  HttpMethod
    params {...}
    body   {...}

    @source [params, body]
    db DbMethod

    response {
        status Concrete<Int>
        @source [params, body, db.returns]
        body {...}
    }
}

@main
Api {
    endpoints []Endpoint
}
api-instance.kli
// DB methods
findUser = DbMethod {
    name    "users.findById"
    args    {id Uuid}
    returns {id Uuid, name String}
}

// "id" in keys(params) ✓
getUser = Endpoint {
    path   "/users/{id}"
    method "GET"
    params {id Uuid}

    // inline refinement works for single fields too
    db findUser & {
        id Uuid = params.id
    }

    response {
        status 200
        body {
            id   Uuid   = db.returns.id
            name String = db.returns.name
        }
    }
}

api = Api {
    endpoints [getUser]
}

Event Sourcing Example

Domain commands emit events with @source provenance. Tags are associated via @assoc and validated with @constraint.

dcb-board-spec.ilk
// Tag: either a parametrized field or a unique string
Parametrized {_ String}
Unique Concrete<String>
Tag Parametrized | Unique

// Event: open struct + timestamp, can carry Tag associations
@assoc [Tag]
Event {...} & {timestamp Int}

// QueryItem: all listed events must have all listed tags
QueryItem {
    @constraint forall(tags, t =>
        forall(eventTypes, e => e.assoc(t)))
    eventTypes []Event
    tags       []Tag
}

// Command: fields drive emits via @source
Command {
    fields {...}
    @source [fields]
    emits []Event
    query []QueryItem
}

@main
Board {
    commands []Command
}
dcb-board-instance.kli
// Define tags
userIdTag   = Parametrized {userId String}
userNameTag = Parametrized {name String}
commonTag   = Parametrized {x String}
simpleTag   = Unique "simple-tag"

// Events with associated tags
userRegistered = Event<userIdTag, userNameTag, commonTag> {
    id   String
    name String
}

other = Event<commonTag, simpleTag> {
    hello String
}

// Command with @source provenance
registerUser = Command {
    fields {id String, name String, x String}

    // timestamp generated, id/name implicit match
    emits [userRegistered & {timestamp Int*}]

    // query: both events have commonTag ✓
    query [{
        eventTypes [userRegistered, other]
        tags       [commonTag]
    }]
}

board = Board {commands [registerUser]}

Base Types & Literals

schema.ilk
// Base types
name      String
id        Uuid
count     Int
rate      Float
active    Bool
data      Any              // anything
created   Date
timestamp Timestamp
price     Money

// Constraint levels
label     Concrete<String>  // kli picks one
version   1                 // exact match
instance.kli
✓ Valid
✗ Invalid
name      String           // open type
id        Uuid
count     Int
rate      Float
active    Bool
data      "text"           // any type works
created   Date
timestamp Timestamp
price     Money

label     "webhook"        // concrete literal
version   1                // exact match
name      Int              // ERROR: wrong type
label     String           // ERROR: needs literal
version   2                // ERROR: must be 1

Value Constraints

Three levels: Open (any value), Concrete (kli picks one), Schema-fixed (exact value). Types must match exactly — kli cannot narrow String to "hello".

schema.ilk
// Open: any string
name String

// kli-fixed: kli picks one
label Concrete<String>

// Schema-fixed: must be exactly 1
version 1
instance.kli
✓ Valid
✗ Invalid
name    String       // open — just type
label   "webhook"    // concrete — kli picks value
version 1            // schema-fixed — exact match
name    Int       // ERROR: wrong type
label   String    // ERROR: needs literal, not open type
version 2         // ERROR: must be exactly 1

Structs

{} = empty, {_} = exactly 1 field, {_ String} = 1 field typed, {_, _} = exactly 2, {...} = any number

schema.ilk
// cardinality patterns
Empty {}                        // zero fields
TagField {_}                   // exactly 1 field
TypedTag {_ String}            // 1 field, typed
Pair {_, _}                    // exactly 2 fields
TypedPair {_ Int, _ String}   // 2 fields, typed

// open + required
Event {...} & {timestamp Int}

// specific fields
User {id Uuid, name String}
instance.kli
✓ Valid
✗ Invalid
empty = Empty {}
tag = TagField {userId String}
typed = TypedTag {x String}
pair = Pair {a Int, b Bool}
tpair = TypedPair {x Int, y String}

evt = Event {
    foo       String
    timestamp Int
}

user = User {
    id   Uuid
    name String
}
// ERROR: {_} requires exactly 1 field
tag = TagField {
    a String
    b String  // too many fields
}

// ERROR: {_ String} needs String type
typed = TypedTag {x Int}  // must be String

// ERROR: missing required timestamp
evt = Event {
    foo String
}  // missing: timestamp

Lists

[]T = zero or more, [N]T = exactly N elements

schema.ilk
events []Event    // 0+ events
tags   [3]Tag     // exactly 3 tags
instance.kli
✓ Valid
✗ Invalid
events [evt1, evt2]
tags   [tag1, tag2, tag3]
events [evt1]  // OK: []T allows any count
tags   [tag1, tag2]  // ERROR: [3]T needs exactly 3

Blocks & Optional Fields

ilk: all fields optional by default. kli: ? marks fields that can't be relied upon

schema.ilk
User {
    id    Uuid
    name  String
    email String
}
// all fields optional by default
// presence enforced by @source
instance.kli
✓ Valid
✗ Invalid
fields {
    id     String
    email? String  // optional — don't rely on it
}

// OK: optional target ← optional source
emits [evt & {
    email? String = fields.email
}]
fields {
    email? String  // optional
}

// ERROR: required relies on optional
emits [evt & {
    email String = fields.email
}]  // fields.email is optional

@doc Annotation

kli-only: hints for runtime implementation (not in ilk)

schema.ilk
// No ilk equivalent — kli only
instance.kli
@doc "multiply qty * unitPrice"
totalAmount Int = compute(fields.qty, fields.unitPrice)

@doc "generate UUID v4 at runtime"
correlationId Uuid*

References

&T = reference to a binding of type T (unquoted name, not string). No data flows through references — they identify which binding, not instantiate it.

schema.ilk
Query {
    eventTypes []&Event  // refs to Event bindings
}
instance.kli
✓ Valid
✗ Invalid
cartCreated = Event {...}
itemAdded   = Event {...}

q = Query {
    eventTypes [cartCreated, itemAdded]
}
// ERROR: quoted string instead of binding
q = Query {
    eventTypes ["cartCreated"]  // must be unquoted
}

// ERROR: non-existent binding
q2 = Query {
    eventTypes [unknownEvent]  // not defined
}

Intersection Types

A & B = must satisfy both; right side wins on conflict

schema.ilk
// open struct + required timestamp
Event {...} & {timestamp Int}

// conflict: right side wins
// {id Uuid} & {id String} → {id String}
instance.kli
✓ Valid
✗ Invalid
evt = Event {
    name      String
    extra     Bool
    timestamp Int  // required by &
}
// ERROR: timestamp has wrong type
evt = Event {
    name      String
    timestamp String  // must be Int
}

Union Types

A | B = exactly one variant; discriminated by name

schema.ilk
// Named block branches
Success { value Bool }
Error   { message String }
Response Success | Error

// Identifier-only variants
Status Pending | Active | Archived

// String literal union
HttpMethod "GET" | "POST" | "PUT"
instance.kli
✓ Valid
✗ Invalid
r1 = Success { value true }
r2 = Error { message String }

s = Active

method "POST"
// ERROR: unknown variant
r = Unknown { x Int }  // not Success|Error

// ERROR: invalid literal
method "DELETE"  // not in "GET"|"POST"|"PUT"

Associated Values

@assoc [T] = instances can carry references to T bindings in angle brackets. Not generics — just metadata attachments for @constraint predicates like e.assoc(t).

schema.ilk
// Tag variants (see Union section)
Parametrized {_ String}
Unique Concrete<String>
Tag Parametrized | Unique

@assoc [Tag]
Event {...} & {timestamp Int}
instance.kli
✓ Valid
✗ Invalid
userIdTag = Parametrized {userId String}
commonTag = Parametrized {x String}

// with associations
evt = Event<userIdTag, commonTag> {
    id        String
    timestamp Int
}

// no associations — omit brackets
other = Event {
    x         String
    timestamp Int
}
// ERROR: wrong type in assoc
notATag = SomeOther {...}

evt = Event<notATag> {...}
// notATag is not a Tag

// ERROR: empty brackets not allowed
evt2 = Event<> {...}  // omit instead

Field Origins (@source)

@source [fields] constrains where data can come from. Each target field must trace to a source: Type* = generated (exempt), = path = mapped, = compute(...) = derived from multiple sources.

schema.ilk
Command {
    fields {...}

    @source [fields]
    emits []Event
}
instance.kli
✓ Valid
✗ Invalid
cmd = Command {
    fields {
        id    String
        name  String
        qty   Int
        price Int
    }

    emits [userRegistered & {
        timestamp Int*           // generated
        id        String         // implicit match
        custId    String = fields.id  // mapped
        total     Int = compute(fields.qty, fields.price)
    }]
}
// ERROR: String cannot narrow to Uuid
cmd = Command {
    fields {
        id   String
    }

    emits [userRegistered & {
        id Uuid  // String 
    }]
}

Subtyping

Closed structs require exact match. Lists and references are covariant.

schema.ilk
// Closed struct — exact fields
Point {x Int, y Int}

// Open with required — width subtyping ok
Event {...} & {timestamp Int}

// Lists covariant
events []Event

// References covariant
ref &Event
instance.kli
✓ Valid
✗ Invalid
// Exact match for closed struct
p = Point {x Int, y Int}

// Open struct: extra fields ok
evt = Event {
    name      String
    timestamp Int
}

// List accepts Event subtypes
events [evt]

// Ref accepts Event subtype
ref evt
// ERROR: extra field in closed struct
p = Point {
    x Int
    y Int
    z Int  // not allowed
}

// ERROR: missing field in closed struct
p2 = Point {
    x Int
}  // missing: y

Constraints

@constraint with forall, exists, unique, count

schema.ilk
QueryItem {
    @constraint forall(tags, t =>
        forall(eventTypes, e => e.assoc(t)))

    eventTypes []Event
    tags       []Tag
}

// More constraint examples:
@constraint unique(items, i => i.id)
@constraint count(items) >= 1
@constraint exists(events, e => e.priority)
instance.kli
✓ Valid
✗ Invalid
// commonTag is associated with both events
evt1 = Event<commonTag> {...}
evt2 = Event<commonTag> {...}

q = QueryItem {
    eventTypes [evt1, evt2]
    tags       [commonTag]
}  // all events have all tags
// ERROR: evt2 not associated with tag1
evt1 = Event<tag1> {...}
evt2 = Event<tag2> {...}

q = QueryItem {
    eventTypes [evt1, evt2]
    tags       [tag1]
}  // evt2.assoc(tag1) = false
